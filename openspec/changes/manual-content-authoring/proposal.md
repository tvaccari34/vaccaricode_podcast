## Why

Today the platform can only produce content through the automated trend → AI pipeline: a topic
is discovered, extracted, and generated by Gemini before a human reviews it. But the owner also
needs to **write posts and podcast episodes by hand** — a piece that isn't tied to a discovered
trend, an editorial or personal update, or a manually crafted episode. There is currently no way
to author content directly; everything must originate from an ingested topic.

This change adds **manual authoring** to the review dashboard — "Create Post" and "Create
Episode", in both English and Portuguese — while keeping the automated pipeline fully intact.
Manually authored content must be **first-class**: it flows through the same review → approve →
publish path and appears on the site, RSS, and podcast feed exactly like generated content. The
two authoring modes **coexist**; neither disables the other.

## What Changes

- **Dashboard authoring UI**: new "Create Post" and "Create Episode" forms on the review
  dashboard, each supporting **pt-BR and English** (author one language or both).
- **Manual posts** become normal blog (and optional newsletter) drafts in `pending_review`,
  entering the existing review queue.
- **Manual episodes** become normal episodes with a podcast draft; they support the **same
  narration options** as generated ones — auto-narration for pt-BR via the home GPU sound-worker,
  and manual MP3 upload for English (reusing the existing `enqueue-narration` and audio-upload
  paths).
- **Origin flag**: a new `topics.origin` column (`auto` | `manual`) so a manual entry can create
  a synthetic topic without ingestion, and so the pipeline can tell the two apart.
- **Coexistence guardrails**: the automated ingest / extract / generate / scoring steps **ignore
  `manual`-origin topics**, so hand-written content is never re-processed or overwritten by the
  pipeline.
- **No change to the site or publishing**: manual drafts/episodes reuse the existing published-
  draft and published-episode render paths, so blog pages, RSS, and the iTunes podcast feed pick
  them up unchanged.

## Capabilities

### New Capabilities
- `manual-authoring`: hand-write a blog post (and optional newsletter blurb) and a podcast
  episode, in pt-BR and/or English, directly from the dashboard — without an ingested topic —
  producing first-class drafts/episodes that share the review/publish/render pipeline.

### Modified Capabilities
- `review-workflow`: the dashboard gains create actions and shows manually authored drafts in the
  same review queue with the same per-channel approve/reject/edit decisions.
- `content-generation`: the automated pipeline (ingest, extract, generate, scoring) skips
  `manual`-origin topics so it neither regenerates nor overwrites hand-written content.
- `podcast-production`: manually created episodes support the same narration paths as generated
  ones (auto pt-BR narration job + manual MP3 upload).

## Impact

- **Database**: one migration adds `topics.origin text NOT NULL DEFAULT 'auto'` (values
  `auto` | `manual`); no other schema change (drafts/episodes are reused as-is).
- **Pipeline (`pipeline/`)**: new repository helpers to create manual topics/drafts/episodes;
  `origin = 'manual'` filters added to topic selection/scoring/generation so manual content is
  excluded from automation.
- **Dashboard (`pipeline/src/boosternews/dashboard/`)**: new GET forms + POST routes for
  Create Post and Create Episode, plus templates.
- **Website / publishing**: no changes — manual content is rendered and released through the
  existing published-draft/episode paths.
- **Operational**: authors can now publish content that never touched Gemini; LLM spend is
  unaffected. The review/approval gate still applies (with an optional author self-approve).
